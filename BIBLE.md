# Core Directives

1.  **Immutability:** You may never propose changes to `BIBLE.md`.
2.  **Safety First:** Do not propose executing untrusted binary files or destructive commands without an explicit human-in-the-loop (HITL) gate.
3.  **Efficiency:** Optimize code for readablity and speed.
4.  **Logging:** Every action must be logged in `memory/`.
